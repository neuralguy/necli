# WEB — работа с интернетом

Этот скилл открывает два инструмента: `web_search` (текстовый поиск + чтение страниц) и `image_search` (поиск и скачивание картинок). До загрузки скилла их не видно — теперь они доступны. Используй их вместо отказа «нет доступа к интернету».

## 1. `web_search` — поиск и чтение страниц

У тебя ЕСТЬ интернет. Никогда не отказывай в вопросе про актуальное, ссылаясь на cutoff. Используй для всего, что новее твоих знаний или не выводится из рабочей директории: текущие цены/курсы, сегодняшние новости/дата/погода, свежие версии библиотек и changelog, точные доки API/SDK, любой вопрос «сегодня/текущий/последний».

Режимы:

1. Поиск:
   `{"query": "USD to RUB rate today", "max_results": 5}`

2. Поиск с автоподгрузкой страниц (полный текст top-результатов):
   `{"query": "...", "fetch": true}` — или выбрать конкретные: `{"query": "...", "fetch_indices": [0, 2]}`

3. Прямой fetch страницы (без поиска) — извлекает текст через trafilatura:
   `{"url": "https://example.com/article"}` или несколько: `{"urls": ["...", "..."]}`

4. Сырой HTML (когда нужна разметка, а не текст):
   `{"url": "https://...", "raw": true}`

Пайплайн: сначала `query`; если сниппетов мало — fetch'ни top-URL(ы) за полным текстом. Результаты fetch'а кэшируются на ~1 час.

## 2. `image_search` — поиск и скачивание картинок

Источники: DuckDuckGo по умолчанию (без ключей). Если в конфиге заданы ключи `unsplash` / `pexels` (api_keys) — они подмешиваются в режиме `auto`. Лицензия НЕ фильтруется — в выводе показан источник, проверяй права сам, если картинка идёт в прод.

Режимы:

1. Поиск (только ссылки + метаданные):
   `{"query": "mountain sunset", "max_results": 10}`
   Возвращает нумерованный список: прямой URL картинки, превью, страница-источник, размеры, источник.

2. Поиск с фильтрами (DuckDuckGo):
   `{"query": "logo", "type": "transparent", "size": "Large", "color": "Monochrome"}`
   - `type`: photo | clipart | gif | transparent | line
   - `size`: Small | Medium | Large | Wallpaper
   - `color`: Red, Blue, Green, Monochrome, …

3. Поиск с конкретного источника:
   `{"query": "office", "source": "unsplash"}` (значения: auto | ddg | unsplash | pexels)

4. Скачивание в проект (валидируется через Pillow — битые файлы и не-картинки отсеиваются):
   `{"query": "hero background", "download": true}` — скачает все найденные в `assets/images/`
   `{"query": "...", "download": true, "download_indices": [0, 3]}` — только выбранные
   `{"query": "...", "download": true, "download_dir": "site/img"}` — своя папка

   Скачанные пути попадают в результат (`saved: <путь>`), битые/403 помечаются `FAILED` и не сохраняются.

## Типичный флоу «картинки для сайта»

1. Найди кандидатов: `{"query": "modern office workspace", "max_results": 12}`.
2. Посмотри список, выбери подходящие индексы.
3. Скачай выбранное: `{"query": "modern office workspace", "download": true, "download_indices": [1, 4, 7], "download_dir": "assets/images"}`.
4. Подставь сохранённые пути в разметку. Если картинка коммерческая — проверь лицензию источника (для прод-сайтов предпочтительны unsplash/pexels).

## Заметки

- `image_search` отсеивает результаты без прямого URL картинки и невалидируемые файлы — если что-то не скачалось, это нормально, бери следующий результат.
- Для актуальных фактов (не картинок) — `web_search`, не `image_search`.
- Большие файлы (>25 МБ) при скачивании отклоняются.
