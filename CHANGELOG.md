# Changelog

Все заметные изменения проекта. Формат основан на [Keep a Changelog](https://keepachangelog.com/),
проект следует [семантическому версионированию](https://semver.org/).

## [0.15.0] — 2026-06-23

### Добавлено

- **Автономный режим оркестрации (`master`).** Главный агент может прогонять задачу
  до конца сам, разбивая её на фазы субагентов без участия пользователя. Усилен
  протокол охоты за багами, разрешён `shell` в автономном режиме, добавлена защита
  от галлюцинаций (агент не отчитывается о действии, которого не делал).
- **Авто-проверки Python-правок.** Каждый отредактированный/созданный `.py` фоном
  ставится в очередь на `lsp_diagnostics` + `ruff` (`tools/auto_checks.py`) —
  результат прилетает уведомлением, дубли проверок не нужны.
- **Команда `/proxy`** (`commands/menus/proxy.py`) — настройка прокси прямо из меню.
- **Доставка системного промта первым сообщением** для прокси, переопределяющих
  system role: системник уходит как первое user-сообщение, иначе провайдер его игнорировал.
- **Конфиг `ruff`** в `pyproject.toml`: прагматичный набор правил
  (E/W/F/I/N/UP/B/C4/SIM/RUF/PIE/PERF/ASYNC/LOG/TID), per-file-ignores для
  `tests/`/`scripts/`, секция `format`.
- **`git_stats.py`** — статистика репозитория для статус-строки/проекта.

### Изменено

- **Каталог провайдеров урезан до 8 ключевых** и обновлён прайсинг/модели (июнь 2026):
  добавлен `aerolink`, удалены `cerebras`, `cohere`, `deepseek`, `fireworks`,
  `hyperbolic`, `llamacpp`, `mistral`, `perplexity`, `sambanova`, `together`, `vllm`.
- **Системный промт разбит на модули** (`prompts/_core`, `_interaction`, `_tooling`,
  `_quality`, `_modalities`, `_subagents`, `_workflow`, `_autonomous`): секции сжаты,
  tool-материал собран в единую секцию S5, tool-сигнатуры рендерятся из `TOOL_SCHEMAS`.
- **Сжатие вывода инструментов** — длинные результаты усекаются с возможностью
  раскрыть полностью.
- **Скиллы разделены** на default (в git) и user (в gitignore); `bot-dev` → user.
- **Память:** таймстампы у записей, приоритетные (pinned) факты закрепляются в промте.
- **UI:** белая подсветка выбора в меню, превью темы в онбординге как в `/themes`,
  упрощён вывод токенов в статус-строке, заголовок до 200 символов, фикс ширины меню сессий.
- **Recap короче** (1–2 предложения), уточнён промт режима планирования.

### Исправлено

- **Парсинг fenced-вызовов:** обработка inline-маркеров `:::call`, ужесточено
  правило остановки после вызова, уведомление о зацикливании на одном инструменте,
  устойчивость к мусору в теле результата.
- **`read_files`:** разбор диапазона строк через двоеточие.

## [0.14.0] — 2026-06-16

### Изменено — оркестрация

- **Подсистема workflow удалена.** Инструмент `workflow`, команда `/workflows`,
  Python-оркестратор (`workflows/runner.py`) и связанные state-артефакты убраны.
  Вся оркестрация теперь делается одним инструментом `subagent`: фазы, pipeline и
  fan-out выражаются прямо в его аргументах.
- **Субагенты — фазовая оркестрация в одном вызове.** `subagent` принимает
  `phases: [{name, tasks}, ...]` (фазы исполняются по порядку, агенты внутри фазы
  параллельно) и `items + stages` (pipeline по элементам). Спеки нормализуются в
  `tools/subagent_specs.py`. Каждая задача ОБЯЗАНА иметь `phase` (группа) и `label`
  (короткое имя в живой панели); без них панель показывала безликие `Agents`/`Sub1`.
  Один вызов прогоняет весь конвейер, помечая каждую завершённую фазу зелёной
  галочкой — не нужно звать `subagent` по разу на фазу и ждать.
- **`isolate` по умолчанию `false`.** Субагенты по умолчанию пишут прямо в общую
  рабочую директорию — значит работу надо резать на независимые слайсы (каждый
  субагент владеет своими файлами). `isolate=true` даёт каждому отдельный git
  worktree.

### Добавлено

- **Команда `/insights`** — анализ всех сохранённых сессий в развёрнутый HTML-отчёт
  (стиль usage-аналитики: at-a-glance, области работы, достижения, трение, фичи к
  пробованию, паттерны, AGENTS.md-добавки) на языке интерфейса, плюс извлечение
  durable-фактов в долговременную память. Локальные метрики считаются на месте,
  модель (`api_insights`, чистый контекст) возвращает структурный JSON-разбор.
  Отчёт сохраняется в `.data/insights/report-<ts>.html`.

### Прочее

- LSP: корректно закрывается subprocess-transport и join loop-потока на shutdown.
- `read_files`: двоеточие как разделитель диапазона строк (`lines="10:60"`).
- `/tool_format`: пересборка истории API при переключении.
- Темы: превью в `/themes` и онбординге переведены на текущий compact-формат без рамок.

## [0.13.9] — 2026-06-12

Большой проход по надёжности оркестрации (субагенты + воркфлоу), контексту и
поведению агента. Найдено и починено живыми headless-прогонами и аудитом кода.

### Исправлено — субагенты и контекст

- **Прунинг контекста субагента.** Субагент слал сырой `session.messages` в модель
  каждую итерацию и копил контекст линейно (54 tool-call ≈ 350k токенов). Теперь
  `_pruned_messages()` чистит историю перед каждым вызовом, как главный цикл.
- **Системный промт субагента урезан на ~2.7k токенов/итерацию.** Убраны блоки,
  которые субагенту недоступны/не нужны: S8 WORKFLOWS, S9 SUBAGENTS, S7.3
  ORCHESTRATION, S2/S2.1 (ответ пользователю). Промт 8.9k→6.1k. На реальном
  multi-file прогоне: 137k→88.6k токенов (−35%).
- **Рабочий token-budget backstop.** Гард читал `buffer.total_tokens`, а на основном
  tool-пути `buffer=None` → лимит никогда не срабатывал. Теперь токены копятся на
  раннере. Лимит поднят 350k→1M.
- **budget/iteration-стоп сигналит error.** Исчерпание бюджета/итераций возвращало
  `error=None` → воркфлоу считал недоведённую работу полным успехом. Теперь это
  явная ошибка (сделанный текст сохраняется).
- **Убран sleep-поллинг «соседа».** Субагент спал, ожидая файл от другого субагента
  (которого не было). Запрещён sleep-for-peer; передан `wave_size` (одиночка знает,
  что пиров нет); `progress.md`/scratchpad — read-once, never poll.
- **Таймаут на вызов модели.** Повисший стрим провайдера больше не блокирует
  субагента навсегда (`asyncio.wait_for`, 240с).

### Исправлено — воркфлоу

- **error-статус при упавших агентах.** При `fail_fast=False` упавшие агенты молча
  записывались, инструмент возвращал `exit_code=0` — главный агент видел зелёный
  результат на провале. Теперь error + список упавших.
- **Финализация state при Ctrl-C/отмене.** `run()` ловил только `Exception`;
  `KeyboardInterrupt`/`CancelledError` оставляли ран навсегда в `running`. Теперь
  `except BaseException` финализирует все running-фазы/агентов.

### Исправлено — инструменты и вывод

- **Относительные пути резолвятся от `--workdir`.** ContextVar рабочей директории не
  переносился в поток `run_in_executor` → пути резолвились от cwd процесса. Теперь
  контекст копируется (`copy_context().run`). Касалось read/write/patch/shell.
- **Ретраи: пол на паузу.** Прокси слал `Retry-After: 0` → 8 мгновенных ретраев
  впустую → падение. Минимальная пауза 1.5с.
- **`0.0s` у read/list скрыт.** Мгновенные операции показывали обманчивое «0.0s»;
  write/patch показывали `0.0s` вместо streaming-времени.
- **При падении shell показывается хвост вывода** (текст ошибки), а не голова
  (`[stderr]` + начало traceback).
- **LSP: точная причина недоступности** + подсказка `grep_files` как fallback.
- **Headless `--json`: ошибки идут JSON на stdout** (для CI/cron), а не plain на stderr.

### Изменено — промт агента

- Решение «соло или оркестрация» (S7.3) перед существенной работой; явная просьба
  пользователя о workflow перебивает эвристику.
- Чтение «locate-then-narrow» (grep/LSP → узкий диапазон) вместо «читать файлы целиком».
- Библиотека паттернов воркфлоу (pipeline=дефолт, barrier=исключение, fan-out→verify,
  adversarial verify, loop-until-dry).
- Убраны дубли поведенческих блоков (S3.5 слит в S3/S7.1).
- Честно про `isolate`: спасает от затирания, но merge одного региона всё равно
  конфликтует — предпочитать distinct-файлы.
- Все скиллы видны в каталоге (снят `disable-model-invocation`); описания, не только имена.
